# ISS-002: Wizard does not update `skillet.skillDir` after moving files to `skill/`

**Severity:** 🔴 Hard fail (install breaks without manual fix)
**Step:** 5 — Install skill
**Discovered:** 2026-06-14, jzOcb/writing-style-skill run (T2)

## Description

When the `create-skillet` wizard moves source files into a `skill/` subdirectory (the file-move step), `package.json`'s `skillet.skillDir` is not updated. It remains `"./"` (the pre-move default) instead of being updated to `"skill/"`.

The install step (`npx . install`) then fails because it looks for `SKILL.md` at the package root (`./`), where the file no longer exists — it was moved to `skill/SKILL.md`.

## Repro

1. Run `npx create-skillet` in a repo where SKILL.md is at the root (e.g. `jzOcb/writing-style-skill`)
2. Accept the default skill content path `"./"` at the wizard prompt
3. When the file-move prompt appears, select SKILL.md (and any other files to include)
4. Confirm — files are moved to `skill/`
5. Run `npx . install` — fails with `SKILL.md not found in <package-root>`
6. Inspect `package.json` — `skillet.skillDir` is still `"./"`

## Expected

After the file-move step, `skillet.skillDir` in `package.json` should be updated to `"skill/"` to reflect where the files were placed.

## Workaround

Manually run `npm pkg set skillet.skillDir=skill/` after packaging. Then also see ISS-003.
