# ISS-003: `bin/cli.js` ignores `skillet.skillDir` and uses package root as skill directory

**Severity:** 🔴 Hard fail (install broken even after fixing `skillet.skillDir`)
**Step:** 5 — Install skill
**Discovered:** 2026-06-14, jzOcb/writing-style-skill run (T2)

## Description

`bin/cli.js` passes the package root directory as `skillDir` to `run()`, ignoring the `pkg.skillet.skillDir` value from `package.json`. This means even after manually fixing `skillet.skillDir` to `"skill/"` (workaround for ISS-002), the install still fails — `run()` still looks for SKILL.md at the package root.

The fix requires patching `bin/cli.js` to compute `skillDir = join(pkgRoot, pkg.skillet?.skillDir)` before passing it to `run()`.

## Repro

1. Complete packaging via `create-skillet` with files moved to `skill/`
2. Manually fix `skillet.skillDir` to `"skill/"` (ISS-002 workaround)
3. Run `npx . install` — still fails with `SKILL.md not found`
4. Inspect `bin/cli.js` — `run()` receives package root, not the `skill/` subdirectory

## Expected

`bin/cli.js` should read `pkg.skillet.skillDir` and join it with the package root to determine the skill directory, e.g.:

```js
const skillDir = join(pkgRoot, pkg.skillet?.skillDir ?? '.');
```

## Workaround

Manually patch `bin/cli.js` to compute `skillDir` from `pkg.skillet.skillDir`. After this fix, `npx . install` works correctly.

## Relationship to ISS-002

Both ISS-002 and ISS-003 must be fixed for T2+ repos to work end-to-end. ISS-002 is the wizard not writing the correct `skillet.skillDir`; ISS-003 is the generated `bin/cli.js` not reading it even if correctly set.
