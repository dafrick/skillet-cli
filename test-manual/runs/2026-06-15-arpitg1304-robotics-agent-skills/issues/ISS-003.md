# ISS-003: bin/cli.js hardcodes skill directory path as absolute URL

**Severity:** Medium (path breaks if package is moved or published)
**Discovered:** 2026-06-15, arpitg1304/robotics-agent-skills test run

## Symptom

The generated `bin/cli.js` contains:

```js
await run({ skillDir: fileURLToPath(new URL('../skills/docker-ros2-development/', import.meta.url)), pkg });
```

The skill directory name `skills/docker-ros2-development/` is hardcoded as a literal string in the generated file. This means:
- If the package is renamed or the skill directory is renamed, the bin script breaks without updating cli.js
- The path is redundant with `package.json`'s `skillet.skillDir` field which already contains the same value

## Expected Behavior

The `bin/cli.js` should derive the skill path from `package.json` at runtime rather than hardcoding it, e.g.:

```js
await run({ pkg }); // let @skillet-cli/core read skillDir from pkg.skillet.skillDir
```

Or at minimum, read `skillDir` from the imported `pkg` object rather than embedding it in the source.

## Context

Generated `package.json` `skillet` field:
```json
"skillet": {
  "skillDir": "skills/docker-ros2-development/"
}
```

Generated `bin/cli.js`:
```js
await run({ skillDir: fileURLToPath(new URL('../skills/docker-ros2-development/', import.meta.url)), pkg });
```

The same path appears in both places — single source of truth is violated.
