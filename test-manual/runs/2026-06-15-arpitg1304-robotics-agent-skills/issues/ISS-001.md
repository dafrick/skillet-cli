# ISS-001: `npm pkg set` fails when description is empty

**Severity:** High (blocks packaging)
**Discovered:** 2026-06-15, arpitg1304/robotics-agent-skills test run

## Symptom

Wizard crashes at "Seasoning package fields" step with:

```
npm error code EUSAGE
npm error npm pkg set expects a key=value pair of args.
✗ Setup failed
Error during setup: npm pkg set exited with code 1
```

## Reproduction

1. Run `npx create-skillet@0.1.3` in a repo with no existing `package.json`
2. Leave the Description prompt empty (just press Enter)
3. Complete remaining prompts
4. Accept "Proceed?"
5. Wizard runs `npm init -y` successfully, then fails on `npm pkg set`

## Root Cause (suspected)

When description is empty string, the wizard passes `description=` (empty value) to `npm pkg set`, which npm rejects as invalid syntax. The command becomes `npm pkg set description=` with no value.

## Expected Behavior

Either:
- Skip setting fields that have empty values, OR
- Validate description is non-empty before proceeding, OR
- Quote the value properly so `npm pkg set description=""` works

## Context

- Repo: `/repo/arpitg1304-robotics-agent-skills`
- No pre-existing `package.json`
- The repo contains 10 skill directories under `skills/`
- The partially-created `package.json` was left on disk after the failure (from `npm init -y`) but without skillet fields set
