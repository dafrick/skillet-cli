# ISS-001: "files" in generated package.json only includes one skill; 9 of 10 skills missing from npm publish manifest

**Severity:** High (publish-time data loss: 9 of 10 skills silently omitted)
**Discovered:** 2026-06-15, arpitg1304/robotics-agent-skills test run (create-skillet v0.2.1)

## Symptom

After running `npx create-skillet@0.2.1` on a 10-skill multi-skill repo, the wizard correctly detects all 10 skills and sets `"skillet": {"skills": "skills"}` in `package.json`. However, the `"files"` field only lists the first skill directory:

```json
"files": [
  "bin",
  "skills/docker-ros2-development/"
]
```

The other 9 skill directories (`skills/robot-bringup/`, `skills/robot-perception/`, etc.) are absent from `"files"`. Confirmed with `npm pack --dry-run`:

```
LICENSE
README.md
bin/cli.js
package.json
skills/docker-ros2-development/SKILL.md
```

Only `skills/docker-ros2-development/SKILL.md` would be included in a published npm tarball. All other skills are silently dropped.

## Impact

- Local `npx . install` works correctly (reads from filesystem, ignores `"files"`)
- `npm publish` + remote install would deliver only 1 of 10 skills to end users
- The bug is invisible during local testing — only manifests after publishing

## Reproduction

1. Use a repo with multiple SKILL.md files under a `skills/` parent directory (e.g. 10 dirs each with SKILL.md)
2. Run `npx create-skillet@0.2.1`
3. Accept multi-skill mode ("Package all skills into one npm package? Yes")
4. Complete wizard
5. Inspect generated `package.json`: `"files"` contains only the first skill directory
6. Confirm with `npm pack --dry-run`: only first skill's SKILL.md appears in tarball

## Expected Behavior

`"files"` should include all skill directories:

```json
"files": [
  "bin",
  "skills/docker-ros2-development/",
  "skills/robot-bringup/",
  "skills/robot-perception/",
  "skills/robotics-design-patterns/",
  "skills/robotics-security/",
  "skills/robotics-software-principles/",
  "skills/robotics-testing/",
  "skills/ros1/",
  "skills/ros2/",
  "skills/ros2-web-integration/"
]
```

Or alternatively, include the entire parent directory: `"skills/"`.

## Context

- Repo: `arpitg1304/robotics-agent-skills` (10 skills under `skills/`)
- `skillet.skills` = `"skills"` (correct)
- `files` = `["bin", "skills/docker-ros2-development/"]` (only first skill)
- create-skillet v0.2.1, @skillet-cli/core 0.3.1
