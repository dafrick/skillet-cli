# ISS-002: Multi-skill repos not detected — single-select forces choice of one skill

**Severity:** High (10 skills present, only 1 gets packaged)
**Discovered:** 2026-06-15, arpitg1304/robotics-agent-skills test run

## Symptom

The repo contains 10 skill directories under `skills/`. The wizard does not detect this as a multi-skill repository. Instead of entering a multi-skill packaging mode (or offering a checkbox to select all), it presents a single-select list: "Select skill content path (relative to package root)". The user must pick exactly one.

Result: only `skills/docker-ros2-development/` (the first in the list) is packaged. The other 9 skills (robot-bringup, robot-perception, robotics-design-patterns, robotics-security, robotics-software-principles, robotics-testing, ros1, ros2, ros2-web-integration) are silently ignored.

## Reproduction

1. Use a repo with multiple SKILL.md files under a `skills/` parent directory (e.g. 10 dirs each with SKILL.md)
2. Run `npx create-skillet@0.1.3`
3. After License prompt, observe: single-select list "Select skill content path", not multi-select checkbox

## Expected Behavior

With 10 skill directories detected, the wizard should either:
- Automatically enter "multi-skill mode" that packages all skills, OR
- Show a multi-select checkbox list ("Select skill directories to include") defaulting to all checked, OR
- Ask "This repo contains 10 skills. Package all of them? (Y/n)"

## Context

- Repo: `/repo/arpitg1304-robotics-agent-skills`
- 10 skills: docker-ros2-development, robot-bringup, robot-perception, robotics-design-patterns, robotics-security, robotics-software-principles, robotics-testing, ros1, ros2, ros2-web-integration
- No root `package.json` before running wizard
- `skillsParent` concept not invoked; wizard uses `skillDir` (singular) throughout
