# ISS-002: Multi-skill install presents separate scope+target wizard for each skill (10 prompts for 10 skills)

**Severity:** Medium (UX friction — functional but exhausting for large multi-skill repos)
**Discovered:** 2026-06-15, arpitg1304/robotics-agent-skills test run (create-skillet v0.2.1)

## Symptom

When installing a multi-skill package with `npx . install`, the installer presents a separate interactive wizard (scope prompt + target selection prompt) for each individual skill. For a 10-skill repo, this requires 20 prompt interactions (10 × scope + 10 × target) to complete the install.

Each wizard cycle looks like:

```
? Install scope:
❯ user   — install for this user account (recommended)
  project — install for this project only

? Select targets to install:
❯◯ Claude Code (not detected)
 ◯ GitHub Copilot (not detected)
 ◯ Agents (.agents/) (not detected)
```

After confirming the first skill, the same prompts reappear for the next skill.

## Impact

- For 10-skill repos: 20 prompt interactions required
- For larger repos (T5: 24–54 skills): potentially 100+ interactions
- Users who choose "not detected" targets must re-select for every skill
- No way to say "install all skills with the same settings"

## Reproduction

1. Use a repo with 10 SKILL.md files under `skills/`
2. Run `npx create-skillet@0.2.1` (multi-skill mode)
3. Run `npx . install`
4. Select scope and target for first skill — observe the same prompts appear for the second skill
5. Repeat 10 times

## Expected Behavior

For multi-skill packages, the installer should collect scope and target once, then install all skills in a single batch:

```
? Install scope: user
? Select targets to install: Claude Code

✓ Installed 10 skills to /root/.claude/skills/
  docker-ros2-development  Baked
  robot-bringup            Grilled
  ...
  10 targets installed · 0.4s
```

## Context

- Repo: `arpitg1304/robotics-agent-skills` (10 skills)
- @skillet-cli/core 0.3.1
- Each of the 10 skills ran a separate install wizard cycle
- Local `npx . install` was the test; remote `npm install` + install would have the same UX
