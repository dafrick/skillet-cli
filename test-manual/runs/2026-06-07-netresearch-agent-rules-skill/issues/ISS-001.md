# ISS-001: create-skillet crashes on startup — fonts directory missing from npm package

**Description**

`create-skillet@0.1.1` crashes immediately on startup with `ENOENT: no such file or directory, open '.../create-skillet/fonts/ANSI Shadow.flf'`. The figlet font files bundled in the source repository are not included in the published npm package.

**Steps to reproduce**

1. `npm install -g create-skillet@0.1.1` (or `npx create-skillet@0.1.1`)
2. Run `create-skillet` from any directory

**Expected result**

The wizard launches and prompts for package configuration.

**Actual result**

```
Error: ENOENT: no such file or directory, open '/usr/lib/node_modules/create-skillet/fonts/ANSI Shadow.flf'
```

The process exits immediately.

**How encountered**

Step 3 — running create-skillet (first attempt via npx, confirmed on global install)

**Why it is bad / user impact**

A first-time user following the npm README cannot use the tool at all. There is no indication in the error message that a font file is missing from the package. Without knowing that figlet is the underlying library and that its fonts need to be copied in manually, the user is completely blocked.

**Severity**

- [ ] critical
- [x] high
- [ ] medium
- [ ] low

**Workaround**

Install the `figlet` npm package locally, then copy its fonts directory into the global create-skillet installation:

```bash
npm install figlet
cp -r node_modules/figlet/fonts /usr/lib/node_modules/create-skillet/fonts
```

This requires knowing that `figlet` is the underlying dependency — not discoverable from the error message alone.
