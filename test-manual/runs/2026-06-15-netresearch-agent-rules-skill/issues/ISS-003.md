# ISS-003: bin/cli.js hardcodes skillDir instead of reading from pkg.skillet.skillDir

## Observed

After `create-skillet@0.1.3` ran on `netresearch/agent-rules-skill`, the generated `bin/cli.js` contains:

```js
await run({ skillDir: fileURLToPath(new URL('../skills/agent-rules/', import.meta.url)), pkg });
```

The path `../skills/agent-rules/` is hardcoded at generation time.

## Expected

The cli should read the skillDir from the package.json `skillet.skillDir` field, similar to the workaround applied in a prior session:

```js
const pkgRoot = fileURLToPath(new URL('.././', import.meta.url))
const skillDir = join(pkgRoot, pkg.skillet?.skillDir ?? './')
await run({ skillDir, pkg })
```

## Impact

If `skillet.skillDir` is changed after packaging (e.g., via ISS-002 file-move), `bin/cli.js` will still point to the old path since it ignores the package.json field. In this run, the hardcoded path happened to match so install succeeded.

## Repro

1. Run `npx create-skillet@0.1.3` on any repo where skillDir differs from `./`
2. Inspect generated `bin/cli.js`
3. The URL path is hardcoded, not derived from `pkg.skillet.skillDir`
