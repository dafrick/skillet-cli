# ISS-001 — Empty author/description fields crash create-skillet wizard

**Run:** 2026-06-15-hao0321-claude-skill-code-cleanup
**Step:** Step 3 — Run create-skillet
**Severity:** 🔶 Mid fail — functional issue (workaround available: re-run with non-empty values)

---

## Summary

When the user leaves the description or author fields empty in the `create-skillet` wizard, the wizard crashes with a non-actionable error from npm. The packaging step fails and must be re-run from the beginning with non-empty values.

---

## Steps to Reproduce

1. Run `npx create-skillet@0.2.1` in a repo with no existing `package.json`
2. When prompted for description, press Enter to leave it empty
3. When prompted for author, press Enter to leave it empty
4. Proceed through the remaining wizard prompts

**Result:** The wizard calls `npm pkg set description= author=` with empty-string values. npm rejects this:

```
npm pkg set expects a key=value pair of args
```

`npm init -y` has already run at this point (creating a bare `package.json`), but the skillet-specific fields (`skillet.skillDir`, `bin`, etc.) are not set. The run aborts.

**Expected:** Either the wizard validates non-empty values inline and prompts again, or it skips the `npm pkg set` call for empty fields and leaves the default from `npm init -y`.

---

## Workaround

Re-run `npx create-skillet`. On the second run, the existing `package.json` (from `npm init -y`) causes the wizard to pre-populate the description field. Provide any non-empty string for author. The wizard completes successfully.

---

## Impact

- Affects any first-time user who leaves description or author blank (a natural choice when those fields seem optional)
- Requires a full wizard re-run; no recovery path within the failed run
- Error message ("npm pkg set expects a key=value pair of args") is not user-friendly and does not indicate which field caused the failure
