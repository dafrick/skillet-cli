# ISS-001: Empty author field causes `npm pkg set` error at Seasoning step

**Severity:** 🔶 Mid fail (workaround available)
**Step:** 3 — Run create-skillet
**Discovered:** 2026-06-14, hao0321-claude-skill-code-cleanup run

## Description

Leaving the Author prompt blank during the `create-skillet` wizard causes a hard error at the "Seasoning package fields" step:

```
npm pkg set expects a key=value pair of args
```

The wizard invokes `npm pkg set author=` (empty value), which npm rejects as invalid.

## Repro

1. Run `npx create-skillet` in a skill repo
2. At the Author prompt, press Enter without entering a value (leave blank)
3. Proceed through remaining prompts and confirm
4. Observe error at "Seasoning" step

## Observed output

```
✓ Prepping   done
✗ Seasoning  npm pkg set expects a key=value pair of args
```

## Workaround

Enter any non-empty string for the Author field. The wizard succeeds on a re-run with author supplied.

## Additional notes

The bug may also interact with non-ASCII characters in the description field — a second run with author provided succeeded even though the description contained Chinese characters. The root cause may be in the `npm pkg set` invocation when any field value is empty or contains characters that confuse the argument parsing. Further investigation needed.

## Suggested fix

Guard the wizard against empty Author submissions, or omit `npm pkg set author=` when the value is empty (fall back to the `npm init -y` default or leave the field unset).
